# 项目经历 · CollabAI Studio

> **CollabAI Studio** — 多人实时协作在线工作台  
> 技术栈：Next.js · React · TypeScript · Tiptap · TLDraw · Y.js · Liveblocks · Prisma · IndexedDB · NextAuth

---

## 🌟 项目亮点（STAR 法则）

### 1. 无冲突实时协同编辑引擎

| | |
|---|---|
| **Situation** | 多人同时编辑同一文档时，传统"最后写入者胜"策略会导致内容互相覆盖，严重破坏协作体验。 |
| **Task** | 实现任意并发操作下的零冲突合并，同时保持光标同步与低延迟。 |
| **Action** | 引入 Y.js（CRDT 算法）作为底层数据一致性层，对接 Liveblocks 进行网络广播；文档模式深度集成 Tiptap，因官方 CollaborationCursor 存在兼容性问题，直接封装 y-tiptap 底层自定义实现多人光标；白板模式为 TLDraw 设计 `useYjsStore` 数据结构转换层，将原生状态桥接至 Y.js。 |
| **Result** | 两人同时修改同一行文字可自动合并，不发生覆盖；多端光标位置实时可见，延迟稳定在网络级别。 |

---

### 2. 离线优先架构与多 Tab 数据同步

| | |
|---|---|
| **Situation** | 网络不稳定时协作工具完全不可用，且浏览器多标签页之间的本地数据相互孤立，无法保持一致。 |
| **Task** | 实现断网后仍可完整编辑，恢复网络后数据无缝上传，同时保证同一浏览器多 Tab 的本地数据实时同步。 |
| **Action** | 使用 `idb` 封装 IndexedDB，设计三个独立数据库（`doc` 文档、`editor` 画布、`list` 索引）；自研 `BroadcastChannelProvider`，结合 `y-indexeddb` 监听并广播多 Tab 的 Y.js Doc 变更；网络恢复后先通过本地数据覆盖内存状态，再经由 Liveblocks 触发增量上传。 |
| **Result** | 断网状态下可正常新建、编辑文档及工作流；重新联网后数据自动同步，无需手动刷新，多 Tab 内容保持一致。 |

---

### 3. 批量元素协作移动的性能优化

| | |
|---|---|
| **Situation** | 白板模式下多人协作时，若用户拖拽大量元素，每帧都将完整的坐标数据广播给其他端，极易造成卡顿与数据风暴。 |
| **Task** | 在保证最终一致性的前提下，大幅降低高频移动操作的网络与计算开销。 |
| **Action** | 设计"相对增量"策略：拖拽开始时广播所选元素的 ID 集合，移动过程中以 60fps 只广播鼠标偏移量（Δx, Δy）；其他用户端渲染时在原坐标上叠加偏移量实现"视觉欺骗"，仅在鼠标松开时才对底层 Y.js 数据做一次真正的坐标结算。 |
| **Result** | 批量移动场景下的广播数据量从 O(n × 坐标数据) 降至 O(1)，多人协作流畅度显著提升，最终数据状态完全一致。 |

### 4. 智能双模式编辑器与图片处理

| | |
|---|---|
| **Situation** | 传统协作工具只提供单一编辑形态，无法同时满足结构化文档写作与自由画布头脑风暴的双重需求；粘贴外部图片时频繁出现防盗链、跨域、内存膨胀等问题。 |
| **Task** | 构建文档与白板双模式无缝切换的工作台，并实现健壮的图片粘贴处理机制。 |
| **Action** | 文档模式基于 Tiptap（ProseMirror）实现块级编辑，支持 H1–H3、代码块、列表、动态卡片及 Markdown；白板模式使用 TLDraw 的 Canvas + React 混合渲染，封装自定义形状组件；通过路由参数 `?mode=page/edgeless` 统一切换。图片粘贴方面，自研 `usePasteImageToTldraw`：网络图片通过 URL 后缀智能判断是否为图片；本地图片上传至后端后使用返回 URL 而非 base64，避免 Y.js 内存膨胀；使用 `new Image()` 获取宽高，`try/catch` 捕获跨域错误并展示"图片失效"占位元素。 |
| **Result** | 一套工作台同时覆盖写作与画布场景，支持 PDF / 图片 / Word 格式导出；图片粘贴兼容网络图片、本地图片、防盗链及跨域等边缘情况，用户体验无缝。 |
