## 项目经历

### CollabAI Studio · 多人实时协作在线工作台

**项目简介**  
一款支持多人实时协作的 Web 工作台，提供文档（富文本块级编辑）与白板（无限画布）双模式，内置 AI 聊天助手，具备完整的离线编辑与多标签页同步能力。

**技术栈**：Next.js · React · TypeScript · Tiptap · TLDraw · Y.js · Liveblocks · Prisma · IndexedDB · NextAuth

---

**① 无冲突实时协同编辑**

- **S**：多人并发编辑同一文档时，传统"最后写入者胜"策略会导致内容互相覆盖，严重破坏协作体验。
- **T**：实现任意并发操作下零冲突合并，同时保证多端光标实时同步。
- **A**：引入 Y.js（CRDT 算法）作为数据一致性层，对接 Liveblocks 进行网络广播；文档模式因官方 `CollaborationCursor` 存在版本兼容性问题，直接封装 `y-tiptap` 底层自实现多人光标；白板模式设计 `useYjsStore` 将 TLDraw 原生状态桥接至 Y.js。
- **R**：两人同时修改同一行文字可自动合并，不发生覆盖；多端光标位置实时可见，延迟稳定在网络级别。

---

**② 离线优先架构与多 Tab 数据同步**

- **S**：网络不稳定时工具完全不可用，且同一浏览器多标签页的本地数据相互孤立。
- **T**：断网后仍可完整编辑，联网后数据无缝上传，多 Tab 内容实时一致。
- **A**：使用 `idb` 封装 IndexedDB，设计三个独立数据库（`doc` 文档、`editor` 画布、`list` 索引）；自研 `BroadcastChannelProvider`，结合 `y-indexeddb` 监听并广播多 Tab 的 Y.js Doc 变更；网络恢复后先本地覆盖内存状态，再经 Liveblocks 增量上传。
- **R**：断网可正常新建、编辑文档；重新联网后数据自动同步，无需手动刷新，多 Tab 内容保持一致。

---

**③ 批量元素协作移动的性能优化**

- **S**：白板模式下拖拽大量元素时，若每帧广播全量坐标数据，极易造成卡顿与数据风暴。
- **T**：在保证最终一致性的前提下，大幅降低高频移动操作的网络开销。
- **A**：设计"相对增量"协议：拖拽开始时广播所选元素 ID 集合，移动过程中以 60 fps 只广播鼠标偏移量（Δx, Δy），其他端在原坐标上叠加偏移量进行视觉渲染；鼠标松开时才对底层 Y.js 数据做一次真正的坐标结算。
- **R**：批量移动场景下广播数据量从 O(n × 坐标数据) 降至 O(1)，协作流畅度显著提升，最终数据状态完全一致。

---

**④ 双模式编辑器与智能图片粘贴**

- **S**：单一编辑形态无法同时满足结构化写作与自由画布需求；外部图片粘贴时频繁出现防盗链、跨域、base64 内存膨胀等问题。
- **T**：构建文档与白板双模式无缝切换的工作台，并实现健壮的图片粘贴机制。
- **A**：文档模式基于 Tiptap（ProseMirror）实现块级编辑，支持 H1–H3、代码块、列表、动态卡片；白板模式使用 TLDraw Canvas + React 混合渲染，通过路由参数 `?mode=page/edgeless` 统一切换；自研 `usePasteImageToTldraw`：网络图片通过 URL 后缀智能判断类型，本地图片上传后端后使用返回 URL（避免 base64 膨胀），用 `new Image()` 获取宽高，`try/catch` 捕获跨域并展示"图片失效"占位元素。
- **R**：一套工作台覆盖写作与画布两种场景，支持 PDF / 图片 / Word 格式导出；图片粘贴兼容网络图片、本地图片、防盗链及跨域等边缘情况。
